# v0.7.2 Application Readiness Model Design

## Goal

Improve final artifact quality without editing any sample-specific output. Shiji, Mao, Effective Requirements, Financial Statement, and Poor Charlie remain diagnostic samples only. The optimization target is KiU's general production capability: models, orchestration, and a small number of safety guardrails.

## Governing Principle

Sample-driven diagnosis, model-driven repair, orchestration-driven delivery, rules-only-as-guardrails.

Implementation priority:

1. Improve the capability model.
2. Place the model at the correct pipeline stage.
3. Add only minimal guardrail rules where the model's boundary must be enforced.

Forbidden approach:

- Do not add book-specific fixes.
- Do not patch generated `SKILL.md` artifacts by hand.
- Do not hard-code sample names, book titles, or one-off source paths into production logic.
- Do not optimize proxy score by reducing case difficulty or weakening gate criteria.

## Problem Diagnosis

The v0.7.2 use-state pass fixed boundary behavior, but final artifact review still exposes general capability gaps:

| Diagnostic symptom | Diagnostic sample | General capability gap |
| --- | --- | --- |
| Complex source can generate too narrow an artifact set. | Effective Requirements | Missing coverage model for source structure and output shape justification. |
| Some primary anchors are information-dense but mechanism-weak. | Shiji | Missing mechanism evidence model for primary anchor selection. |
| Transfer questions can be too generic. | Shiji / Mao / Poor Charlie | Missing transfer fit model that derives constraint questions from mechanism structure. |
| Low-temporal principles can be over-pressured by world context if treated mechanically. | Poor Charlie / philosophical prompts | Need non-dilution as a model invariant, not a post-hoc exception. |

## Unified Model

The next quality pass introduces an `Application Readiness Model` with three submodels.

### 1. Coverage Model

Purpose: Decide whether the source's capability structure is adequately represented by published skills, workflow candidates, source notes, and gateways.

Inputs:

- Graph communities and high-density conceptual clusters.
- Candidate seed distribution.
- Workflow-vs-agentic routing evidence.
- Published skills and workflow candidates.

Outputs:

- `coverage_units`: source clusters or capability areas that deserve representation.
- `covered_units`: units represented by a skill, workflow candidate, gateway, or explicit source-layer note.
- `uncovered_units`: units missing representation.
- `narrow_output_justification`: required when output shape is intentionally narrow.
- `coverage_readiness`: `pass | warn | fail`.

Non-goal: Force a fixed number of skills.

### 2. Mechanism Evidence Model

Purpose: Rank evidence by whether it can support a transferable judgment skill.

Mechanism dimensions:

- `actor_present`
- `action_present`
- `constraint_present`
- `consequence_present`
- `counter_signal_present`
- `mechanism_bridge_present`
- `supports_transfer_conditions`
- `supports_anti_conditions`

Outputs:

- `mechanism_density_score`
- `mechanism_density_reason`
- `primary_anchor_allowed`
- `anchor_role`: `primary | supporting | source_context_only`

Non-goal: Ban text forms such as titles, lists, tables, notes, or genealogies by type. They are downranked only when mechanism evidence is weak.

### 3. Transfer Fit Model

Purpose: Generate specific application-readiness questions when a user tries to transfer a source principle, case, analogy, or mental model to a present situation.

Question dimensions:

- mechanism fit
- incentive mismatch
- institutional/context mismatch
- time horizon mismatch
- resource and authority mismatch
- disconfirming evidence
- anti-condition checks

Outputs:

- `fit_questions`
- `mismatch_questions`
- `disconfirming_evidence_questions`
- `transfer_readiness`: `ready | ask_more_context | refuse`

Non-goal: Add fixed generic questions to every skill. Questions must be derived from the source mechanism and use state.

## Orchestration

```text
After extraction:
  coverage_model identifies source capability coverage needs.

During seed mining / distillation:
  mechanism_evidence_model ranks candidate anchors and blocks mechanism-weak primary anchors.

During application gate generation:
  transfer_fit_model produces concrete transfer-readiness questions for transfer_candidate states.

During evaluation:
  cross-sample readiness review checks coverage, mechanism evidence, transfer fit, non-dilution, and proxy gates.
```

## Guardrails

Rules are allowed only as explicit guardrails:

- `SKILL.md` must not contain world/live fact pollution.
- `current_fact_required` without verification must not direct-apply.
- `workflow_script_candidate` must not be silently promoted into `bundle/skills/`.
- Chapter-title-style candidates with low mechanism evidence must not be published as judgment skills.

Rules must not decide:

- A fixed number of skills per book.
- Book-specific source ranking.
- Book-specific world alignment behavior.
- Fixed transfer questions independent of mechanism.

## Acceptance Gates

### Model Gates

- Coverage model emits auditable `coverage-units.json` and `coverage-readiness.json` for generated runs.
- Mechanism evidence model emits scored anchor candidates and primary-anchor decisions.
- Transfer fit model emits concrete constraint questions for `transfer_candidate` cases.

### Quality Gates

- Five-sample proxy average remains >= 90.
- No `wrong_verdict` or `boundary_leak` in release proxy evidence.
- Final artifact value average reaches >= 92 in internal review.
- Effective Requirements-style breadth caveat is replaced by a coverage readiness judgment.
- Shiji-style weak primary anchors are downranked by mechanism evidence, without hard-coding Shiji.
- Low-risk reflection prompts remain non-diluted.
- Source pollution remains 0.

### Evidence Gates

- The five diagnostic samples are rerun after implementation.
- Release evidence distinguishes model improvements from sample-specific observations.
- Same-source cangjie superiority and external blind review remain unclaimed unless separately proven.
