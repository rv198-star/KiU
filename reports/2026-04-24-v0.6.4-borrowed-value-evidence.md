# KiU v0.6.4 Borrowed Value Maximization Evidence

Date: 2026-04-24

## Scope

v0.6.4 closes the biography/history/case-heavy source gap found during the Shiji cold-start run. The goal is not to make Shiji-specific rules. The goal is a source-shape-driven borrowed-value strategy: maximize transferable use value while rejecting pure source-understanding requests as judgment-skill triggers.

This release remains source-faithful. It does not perform v0.7 world alignment, and it does not ingest external final `SKILL.md` packs into the default production path.

## Implementation Evidence

- `src/kiu_pipeline/source_shape.py` classifies source shape and emits `borrowed_value_maximization` when biography/history/case/argument/policy/action-program signals are present.
- `src/kiu_pipeline/extraction.py` adds graph nodes for `case_mechanism` and `situation_strategy_pattern` with `transfer_conditions`, `anti_conditions`, source coordinates, and agentic priority.
- `src/kiu_pipeline/contracts.py` adds contracts for `historical-analogy-transfer-gate`, `no-investigation-no-decision`, and `principal-contradiction-focus`.
- `src/kiu_pipeline/extraction_bundle.py` renders borrowed-value rationale, evidence summaries, usage notes, scenario families, revision seeds, and eval failure modes.
- `src/kiu_pipeline/seed.py` carries agentic priority into seed scoring and filters degenerate chapter IDs such as pure numerals from publishable candidate IDs.

## Interpretation

v0.6.4 fixes the prior failure mode where a history/case-heavy book could produce only shallow historical judgment output, and where argument-heavy material could bury transferable strategy patterns behind chapter-title candidates.

The most important change is the graph/skill separation:

- Graph/source layer still extracts factual, biographical, chronological, position, and summary evidence.
- Judgment skill layer refuses to trigger on summary, translation, fact lookup, biography intro, author-position query, and stance commentary without a current user decision.

## Known Limits

- The source-shape classifier is heuristic, not LLM-assisted.
- The usage layer is still mostly generated smoke evidence, not external human-written scenario evaluation.
- Mao still produces several chapter-title-derived supplemental candidates. v0.6.4 filters pure numeric headings and prioritizes borrowed-value skills, but broader chapter-title hygiene remains a later quality-improvement item.
- v0.6.4 does not add real-world/current-world alignment. That remains reserved for v0.7.
- v0.6.4 does not use cangjie or Graphify final outputs as hidden generation inputs. External projects remain reference/benchmark material only.
